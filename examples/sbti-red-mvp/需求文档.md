## 1. 背景与目标

### 1.1 业务背景

- 当前计划通过微信群图片广告、小红书内容、二维码海报做低成本拉新。
- 旧版需求只覆盖“测试 + 结果 + 上传”，无法形成真正的传播、留资、复访、变现闭环。
- 本轮需求要把产品调整成“免费即可玩，但天然支持留资、分享、复访和后续付费”的结构。

### 1.2 核心目标

1. 用户可在 3 分钟内完成测试，立即看到带情绪价值的免费报告。
2. 用户可选择留下邮箱接收报告，也可直接跳过，不阻塞测试流程。
3. 系统自动生成一张可分享海报，附人格说明与二维码，帮助裂变回流。
4. 结果页预留 `1.99` 元解锁详细报告能力，并支持微信 / 支付宝 H5 支付接口预留。
5. 运营后台可以看到访问、在线、留资、结果和付费预留数据。

### 1.3 量化指标（首版）

| 指标 | 当前值 | 目标值 | 统计口径 |
|---|---|---|---|
| 测试完成率 | 0 | >= 40% | 完成提交人数 / 打开落地页 UV |
| 海报分享点击率 | 0 | >= 15% | 点击分享/保存海报人数 / 完测人数 |
| 邮箱留资率 | 0 | >= 12% | 留邮箱人数 / 完测人数 |
| 详细报告点击率 | 0 | >= 8% | 点击解锁人数 / 完测人数 |
| 页面可用性 | 无 | >= 99% | 健康检查成功率 |

## 2. 产品定位

- 产品名：`SBTI RED · 红镜人格实验室`
- 核心定位：`可分享、可留资、可回看、可升级的人格测试 H5`
- 价值结构：
  - 免费层：立刻可玩、立刻有反馈、立刻可分享
  - 留资层：邮箱接收报告、后续运营触达
  - 付费层：深度报告解锁

## 5. 核心用户路径

1. 用户扫码进入首页。
2. 首页弹出邮箱采集层：
   - 可直接填写邮箱
   - 可点击“先看看结果”跳过
3. 用户进入昵称确认层：
   - 系统随机生成昵称
   - 可点击“换一个”
   - 可手动输入自定义昵称
4. 用户完成测试并提交。
5. 系统生成免费报告、今日序号、海报图片、分享二维码。
6. 结果页播放轻量烟花特效，展示：
   - 免费报告摘要
   - 保存海报
   - 留邮箱收报告
   - 解锁详细报告
   - 重新测试
7. 用户再次打开时：
   - 如果本地或服务端已有报告缓存，直接进入报告页
   - 仍可点击“重新测试”重新生成

## 6. 功能明细

| 功能点 | 输入 | 处理规则 | 输出 | 异常处理 | 优先级 |
|---|---|---|---|---|---|
| 首次邮箱弹框 | `email / skip` | 默认弹窗一次，允许跳过 | 留资记录或跳过状态 | 邮箱格式错给提示 | P0 |
| 随机昵称 | `session_id` | 按词库随机生成 | 默认昵称 | 允许重新生成 | P0 |
| 自定义昵称 | `nickname` | 用户可覆盖默认值 | 个性昵称 | 长度/敏感词校验 | P0 |
| 测试提交 | 答案集合 | 计算人格编码、维度得分、报告快照 | 免费报告 + reportToken | 校验失败阻止提交 | P0 |
| 免费报告展示 | `reportToken` | 读取缓存快照 | 结果页 | 无缓存时兜底重测 | P0 |
| 海报生成 | 报告快照 | 生成一张海报 PNG + 二维码 | 图片路径、分享素材 | 失败时允许重试 | P0 |
| 报告邮件发送 | `email + reportToken` | 发摘要或访问链接 | 发送成功提示 | 记录失败原因 | P0 |
| 今日第 N 位提示 | 当日完成记录 | 按完成时间排序 | 排名文案 | 查不到则不展示 | P0 |
| 详细报告解锁 | `reportToken` | 显示预览和支付入口 | 锁定页 / 已解锁页 | 未开支付时显示“即将开放” | P1 |
| H5 支付预留 | 支付方式 | 创建订单与回调占位 | order 信息 | 接口返回预留态 | P1 |
| 在线人数统计 | session 心跳 | Redis TTL 计数 | 在线数 | Redis 异常返回 0 | P0 |
| 运营后台 | 统计、筛选、查看详情 | 后台查询 | dashboard / list / detail | 未登录拒绝访问 | P0 |

## 7. 数据与字段规则

| 字段 | 来源类型 | 展示格式 | 是否可编辑 | 校验规则 | 备注 |
|---|---|---|---|---|---|
| `session_id` | 系统生成 | 字符串 | 否 | 非空、长度 <= 64 | 首次访问生成 |
| `report_token` | 系统生成 | 字符串 | 否 | 唯一、长度 <= 64 | 报告分享主键 |
| `nickname` | 随机或用户输入 | 字符串 | 是 | 2-20 字，敏感词过滤 | 可二次修改 |
| `email` | 用户输入 | 邮箱 | 是 | 标准邮箱格式 | 可为空 |
| `result_code` | 系统计算 | 结果编码 | 否 | 必须在人格枚举内 | 支撑报告映射 |
| `poster_path` | 系统生成 | 本地 URL | 否 | 非空 | 指向海报图片 |
| `today_rank` | 系统计算 | 数字 | 否 | >= 1 | 当日完成序号 |
| `unlock_status` | 系统状态 | `locked/unlocked` | 否 | 枚举值 | 详细报告状态 |

## 9. 验收标准

### 9.1 功能验收

- 用户可选择跳过邮箱，不被阻塞。
- 用户可随机昵称、换昵称、手动改昵称。
- 用户提交后能立刻看到免费报告。
- 报告页能显示“你是今天第 N 位测试用户”。
- 系统自动生成海报，并能在结果页查看或保存。
- 用户再次打开能直接看到缓存报告，并可重新测试。
- 后台能看访问、留资、结果详情、在线人数。

### 9.2 体验验收

- H5 页面在移动端单手可操作。
- 结果页视觉有明显“完成感”和“可分享感”。
- 烟花特效不会影响主要按钮点击和滚动。

### 9.3 稳定性验收

- `web/api/mysql/redis` 健康检查通过。
- 邮件发送失败不会阻塞报告主流程。
- 海报生成失败可单独重试，不影响报告查看。

## 10. 风险与回滚策略

| 风险 | 影响范围 | 预防措施 | 回滚动作 |
|---|---|---|---|
| 海报生成耗时过长 | 结果页卡顿 | 改为异步生成 + 轮询 | 先展示纯文本报告 |
| 邮件能力不稳定 | 留资转化体验 | 邮件任务表 + 重试 | 后台手动重发 |
| 支付预留过早上线 | 用户误解可支付 | UI 标注“即将开放” | 关闭支付入口开关 |
| 后台无鉴权 | 数据泄露 | 先做基础登录保护 | 直接下线后台入口 |
