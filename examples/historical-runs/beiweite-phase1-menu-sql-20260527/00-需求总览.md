# 需求总览 — beiweite Phase 1 菜单注册 + DB 修复

## 背景

`beiweite-drawing-system` 是北京贝威特能源的采购智能估价系统 Demo。
后端 Java + ai-service FastAPI + 前端 Vue3 均已完成，但启动后侧边栏没有图纸模块入口，
原因是 `system_menu` 表中缺少图纸模块的菜单数据，且 `price_record.drawing_id` FK 为 NULL。

## 目标

- Phase 1 Done 定义：启动 Demo 后，管理员能在侧边栏看到「采购估价」模块，并正常访问四个子页面。

## 范围

| # | 任务 | 类型 | 优先级 |
|---|---|---|---|
| T1 | 新建 Flyway 迁移 `V1_3__drawing_module_menu.sql` | SQL | P0 |
| T2 | 更新 `drawing-init.sql` 补充 FK 回填 | SQL | P0 |

## 验收标准

- [ ] V1_3 SQL 文件语法正确，可被 PostgreSQL 执行
- [ ] 插入父菜单（采购估价 / 目录型）+ 4 个子菜单（数据概览/工作台/图纸库/价格库）
- [ ] super_admin（role_id=1）获得全部 5 个菜单权限
- [ ] drawing-init.sql 末尾有 UPDATE 语句，执行后 price_record.drawing_id 不再为 NULL
- [ ] setval 语句正确更新 system_menu_seq 和 system_role_menu_seq

## 不在范围

- 前端路由改动（已存在）
- Java 代码改动
- Phase 2 统计仪表盘
