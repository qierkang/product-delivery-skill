## 1. Visual Theme & Atmosphere

- 核心气质：`红镜、轻戏剧性、结果揭晓、可转发`
- 视觉关键词：`雾面白、酒红、墨蓝黑、镜面反光、细边框、轻科技`
- 情绪目标：用户进入首页时先感到“有风格但不压迫”，测试完成后有“被说中”和“想转发”的情绪价值
- 页面密度：首页偏克制，答题页偏聚焦，结果页偏仪式感，后台偏理性
- 用户第一眼应该感知到什么：`这是个精致、可玩、能立刻开始的性格测试，不是杂乱营销页`
- 明确避免什么观感：`大字堆砌、廉价大红大黑、强压迫感、按钮换行、信息拥挤、像模板站`

## 3. Typography Rules

- 标题字体：中文优先 `PingFang SC` / `Hiragino Sans GB`，英文和结果码可混用 `Baskerville` / `Georgia` 风格衬线提升气质
- 正文字体：系统无衬线，保证移动端可读性
- 数字 / 结果码字体：更紧凑、更重，但不要超过容器承载能力
- 字重策略：`700/800` 只用于标题和关键结果；正文尽量 `400/500`
- 标题字距策略：略紧，避免浮夸大字距
- 正文行高策略：`1.6-1.75`

| 层级 | 用途 | 字号建议 | 字重 | 备注 |
|---|---|---|---|---|
| `display` | 首页主标题 | 桌面 `40-52` / 移动 `28-34` | `700` | 不允许连续三行以上巨型标题 |
| `h1` | 页面主标题 | 桌面 `30-36` / 移动 `24-28` | `700` | 结果页可更强 |
| `h2` | 区块标题 | `20-24` | `700` | 保持整齐，不要压内容 |
| `h3` | 卡片标题 | `16-18` | `700` | 输入卡、步骤卡、说明卡 |
| `body` | 正文 | `14-16` | `400/500` | 移动端正文不低于 `14` |
| `meta` | 标签 / 辅助说明 | `11-13` | `600/700` | 尽量单行，不堆太多 |

## 4. Component Stylings

### Buttons

- Primary Button：酒红纯色或极轻渐变，圆角胶囊或圆角矩形，宽度稳定，不换行
- Secondary Button：白底细边框，文字颜色靠近主色，不做厚重阴影
- Ghost Button：透明或极浅底，只做辅助操作
- 禁止：高饱和大面积纯红、按钮内两行文字、过厚投影、渐变过艳

### Cards / Panels

- 主卡片：雾面白 + 轻边框 + 柔阴影，允许一点暖色反射
- 信息卡：留白大于装饰，优先稳定布局
- 数据卡：文字层次明确，数字不要过大压倒标题
- 禁止：每张卡都做强渐变、强玻璃、强描边

### Inputs / Forms

- 输入框风格：白底、细描边、圆角 `16-20`
- 焦点态：主色轻描边 + 柔和 focus ring
- 错误态：就近提示，不用大面积红底
- 禁止：超粗边框、强发光、深色输入框压正文

### Navigation / Chips / Tags

- 顶部导航：轻、薄、可呼吸，不做厚头部
- 标签 / Chip：小而紧凑，优先单行，避免每个 chip 都像按钮
- 分段区块：靠留白和标题分组，不靠大量硬分隔
- 禁止：chip 过大、标签成片堆满首屏

## 5. Layout Principles

- 容器宽度策略：首页主容器控制在 `1040-1120px`，移动端边距 `12-16px`
- 留白策略：用纵向节奏建立高级感，首屏和 CTA 周围留白要足
- 栅格策略：首页桌面双栏，移动端单栏；证明卡桌面三列、移动端一列
- 单屏信息量上限：首页首屏只承载品牌、价值、代号输入、开始测试、图鉴入口
- 首屏重点：`开始测试` 永远是最强操作，`图鉴` 和 `继续上次结果` 是辅助
- 组件之间的节奏：卡片间距 `12-20px`，块间距 `24-40px`

## 6. Depth & Elevation

- 卡片圆角策略：主卡片 `24-32px`，输入容器 `16-20px`
- 阴影策略：大面积柔阴影，不要锐利黑影
- 模糊 / 镜面 / 渐变策略：只作为背景氛围，不能吃掉正文对比度
- 哪些地方可以重、哪些地方必须轻：首页背景气氛可以重，文字区和表单区必须轻

## 7. Do's and Don'ts

### Do

- 用少量酒红和暖铜建立品牌识别
- 让主标题、代号输入和 CTA 形成稳定纵向阅读路径
- 优先保证移动端按钮、输入框、标签都不挤不换行

### Don't

- 不要把首页做成营销长页或文案墙
- 不要让标题字号压过组件与内容本身
- 不要让深色装饰背景影响文字可读性

## 8. Responsive Behavior

- 移动端主断点：`600px`
- 桌面端主断点：`860px` / `1120px`
- 标题缩放规则：小屏优先减小字号，不强行断成过多行
- 按钮换行规则：首页底部按钮默认同层布局，必要时整体纵向堆叠，不允许单按钮内换行
- 卡片折叠规则：桌面双栏内容，移动端全部单列化
- 触控热区要求：按钮和关键可点元素不低于 `44x44`

## 9. Agent Prompt Guide

- 一句话风格提示：`做成一张可直接转发的红镜人格实验室首页，浅底镜面、轻戏剧性、强节奏，不要廉价营销感。`
- 生成页面时必须遵守：`优先可读性、按钮不换行、移动端单手可用、主 CTA 明确、首页不堆信息。`
- 允许自由发挥的范围：`背景雾化、轻反光、细节纹理、徽章和 chip 的处理。`
- 如果视觉与内容冲突，优先级规则：`先保证内容清晰和操作顺畅，再谈风格装饰。`
