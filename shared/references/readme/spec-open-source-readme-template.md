## 使用本模板生成 README 的额外约束

如果你把这份模板交给其他模型使用，建议再附加以下要求：

1. 直接输出完整 README 成品，不输出大纲或分析
2. 全文使用中文简体
3. 保留企业级文档语气，但适配 GitHub 开源读者
4. 功能模块按业务域组织，不按技术名词堆砌
5. 不要只写“如何启动”，必须写清“项目是什么、为什么有用、如何贡献、如何安全反馈、使用边界是什么”
6. 信息不足时可合理补全，但不得编造明显虚假的线上地址、性能数据、社区规模或客户案例
