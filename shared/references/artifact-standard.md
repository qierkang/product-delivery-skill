## 2. 文档元信息硬要求

- 每份文档必须有：`版本`、`日期`、`编写人`、`状态`
- 默认作者统一写：`xyqierkang@gmail（claude+codex）`
- 阶段结论型文档必须有“证据清单”或“验证记录”

## 3. SQL 与数据脚本硬要求

1. 建表语句必须包含中文注释：
   - 表注释（`COMMENT='...'`）
   - 字段注释（`COMMENT '...'`）
   - 索引用途说明（建议 `KEY ... COMMENT '...'`）
2. SQL 文件头必须包含：
   - 作者：`xyqierkang@gmail（claude+codex）`
   - 项目名
   - 脚本用途
3. 设计文档中的表结构描述必须与 SQL 实际字段一致，不能只写抽象字段名。

## 4. 可交接性要求

- 后续接手人不看会话记录，只看仓库文件也能理解：
  - 功能目标是什么
  - 数据怎么设计
  - 接口怎么调用
  - 如何验证与回滚
- 验收产物默认拆成三类：
  - QA 验收
  - UI 验收
  - 产品验收

## 5. 可选扩展模板

以下模板不强制纳入每个项目，但一旦项目需要对应能力，应优先在仓库内落盘，而不是只写在聊天记录里：

1. `ui/page-api-mapping.md`
   - 页面动作与接口映射
   - 适合前后端联调、UI 验收、接口回流
2. `config/MENU_DATA.md`
   - 菜单结构与权限点
   - 适合后台管理类项目
3. `config/DICT_DATA.md`
   - 字典类型与字典值清单
   - 适合存在枚举、状态码、业务字典的项目
4. `config/sql-change.sql`
   - SQL 变更、初始化或回滚脚本
   - 适合存在表结构、初始化数据或菜单数据变更的项目
