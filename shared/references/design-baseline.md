# 设计基线

本文件是 `product-delivery-skill` 在 UI 方案与 UI 验收阶段的设计兜底说明。

若 request 或项目根目录已存在 `DESIGN.md`，则该文件优先级高于本基线，当前文件只承担兜底与校验作用。

## 协同规则

1. 若当前运行端可识别 `ui-ux-pro-max`
   - 优先借助它补设计方向、token、组件策略、动效、可访问性
2. 若当前运行端不可识别该外部 skill
   - 只允许回退到本仓模板和本文件要求
   - 继续完成 UI 方案与 UI 验收，不中断交付
3. 即使运行端还能识别 `frontend-design` 等通用设计 skill，也不得把它们当成默认兜底

## UI 方案最低要求

`UI交互设计规范.md` 至少要明确：

- 是否对齐 `DESIGN.md`
- 风格方向
- 信息架构
- 组件与版式策略
- 动效与反馈
- 多端差异
- 可访问性与响应式要求

## UI 验收最低要求

`UI验收报告.md` 至少要明确：

- 是否符合 `DESIGN.md`
- 页面还原度
- 交互一致性
- 风格方向是否跑偏
- token 是否一致
- 动效是否缺失
- 响应式和可访问性是否有明显问题
